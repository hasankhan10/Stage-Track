Description & Key Fields

Core entity. Fields: id, workspace_id, name, company, email, phone, stage (1–12),

11
Security Requirements
Every item on this list is mandatory. The AI must not skip or simplify any security measure.
  All database access enforced via Supabase Row Level Security (RLS) policies — never bypass RLS
  Admin can only access data within their workspace_id — enforced via RLS
  Team members can only access clients where assigned_to = their user_id — enforced via RLS
  Client progress tokens are UUID v4 — cryptographically unguessable
  Public pages (/progress/[token], /proposals/[token]) are read-only — no write operations possible from
these pages
  File URLs are signed (Supabase signed URLs) with 1-hour expiry — regenerated on each page load
  All API routes validate auth session server-side before returning data
  Environment variables: never expose Supabase service_role key to frontend — use only anon key
client-side
  Rate limiting on /api/client-link and /api/proposal endpoints: 60 requests/minute per IP
  Input sanitization: all text inputs sanitized before DB write (prevent XSS)
